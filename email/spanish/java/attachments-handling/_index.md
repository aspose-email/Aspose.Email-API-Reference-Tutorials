# Extraer archivos adjuntos de correo electrónico con Aspose.Email para Java

En este hub descubrirá todo lo que necesita para **extraer archivos adjuntos de correo electrónico** de los formatos de correo más comunes usando Aspose.Email para Java. Ya sea que esté construyendo un servicio de procesamiento de correo, archivando datos de Outlook, o simplemente necesite extraer archivos de mensajes MSG, EML o PST, estas guías paso a paso le muestran cómo hacerlo de forma rápida y fiable.

## Respuestas rápidas
- **¿Cuál es la forma más fácil de extraer adjuntos de un archivo PST?** Use `PersonalStorage` to open the PST and iterate through `Message` objects, calling `Message.getAttachments()`.  
- **¿Puedo extraer imágenes en línea (embebidas) como archivos separados?** Yes – treat them as regular attachments; Aspose.Email exposes them through the same API.  
- **¿Necesito una licencia para ejecutar los ejemplos?** A temporary license works for development; a full license is required for production.  
- **¿Qué formatos son compatibles para la extracción de adjuntos?** MSG, EML, EMLX, MHTML, y archivos PST son totalmente compatibles.  
- **¿Existe una forma de guardar automáticamente los archivos extraídos?** Absolutely – call `Attachment.save(filePath)` inside a loop to write each attachment to disk.

## ¿Qué es “extraer archivos adjuntos de correo electrónico”?
Extraer archivos adjuntos de correo electrónico significa leer programáticamente un mensaje de correo (o un archivo de buzón) y extraer cualquier archivo que haya sido adjuntado, como documentos, imágenes u objetos incrustados, para que puedan guardarse, procesarse o reenviarse a otro lugar.

## ¿Por qué usar Aspose.Email para Java para extraer archivos adjuntos de correo electrónico?
- **Cobertura completa de formatos** – Funciona con MSG, EML, PST y más sin necesidad de tener Outlook instalado.  
- **Sin interoperabilidad COM** – API pura de Java, ideal para servidores multiplataforma.  
- **Alto rendimiento** – El procesamiento basado en streams le permite manejar buzones grandes de manera eficiente.  
- **Manejo avanzado de adjuntos** – Soporta adjuntos regulares, en línea y codificados en TNEF de forma nativa.

## Requisitos previos
- Java 8 o superior.  
- Biblioteca Aspose.Email para Java (descargue del sitio oficial).  
- Una licencia temporal o completa de Aspose para uso en producción.

## Preguntas frecuentes

**P: ¿Cómo extraigo los adjuntos de correo electrónico de un solo archivo MSG?**  
R: Cargue el archivo con `MailMessage.load("file.msg")` y llame a `mailMessage.getAttachments()`; luego itere y guarde cada adjunto.

**P: ¿Puedo extraer adjuntos de archivos PST cifrados o protegidos con contraseña?**  
R: Sí. Proporcione la contraseña al abrir la instancia `PersonalStorage`: `PersonalStorage.fromFile("file.pst", password)`.

**P: ¿Cuál es la diferencia entre los adjuntos regulares y los adjuntos en línea?**  
R: Los adjuntos regulares son archivos separados, mientras que los adjuntos en línea están incrustados en el cuerpo del correo (a menudo imágenes). Aspose.Email trata a ambos como objetos `Attachment`, permitiendo manejarlos de forma uniforme.

**P: ¿Existe un límite al tamaño de los adjuntos que puedo extraer?**  
R: La biblioteca transmite datos en streams, por lo que solo está limitado por la memoria y el espacio en disco disponibles, no por el tamaño del adjunto.

**P: ¿Necesito cerrar manualmente los streams después de guardar los adjuntos?**  
R: Cuando usa `Attachment.save()`, la biblioteca maneja la liberación del stream automáticamente, pero si abre streams personalizados, recuerde cerrarlos para evitar fugas.
