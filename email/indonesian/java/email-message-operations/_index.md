## Cara mengonversi format email menggunakan Java?
`MailMessage.load` memuat file email (EML, MSG, dll.) ke dalam objek MailMessage. `MailMessage.save` menulis objek ke format baru. Untuk mengonversi, panggil `MailMessage.load` dengan jalur sumber, kemudian panggil `save` dengan menentukan format output yang diinginkan dan tujuan. API menangani enkoding, lampiran, dan metadata secara otomatis.

**Q: Bisakah saya mengonversi file MSG yang dilindungi kata sandi ke EML?**  
A: Ya. Muat pesan dengan parameter kata sandi yang sesuai, kemudian panggil `save` dengan format yang diinginkan; API secara otomatis mendekripsi dan mengenkripsi kembali konten.

**Q: Apakah Aspose.Email untuk Java memerlukan Microsoft Outlook terpasang?**  
A: Tidak. Pustaka ini berfungsi sepenuhnya secara independen dari Outlook atau Exchange Server, menjadikannya ideal untuk konversi batch sisi server.

**Q: Bagaimana cara menangani file PST besar tanpa menghabiskan memori?**  
**`PstReader` menyediakan API streaming untuk membaca file PST item demi item.** Gunakan API streaming `PstReader`, yang membaca item sesuai permintaan dan menulisnya secara bertahap, menjaga penggunaan memori di bawah 100 MB bahkan untuk PST multi‑gigabyte.

**Q: Apakah ada cara untuk mempertahankan cap waktu email asli selama konversi?**  
A: Tentu saja. Objek `MailMessage` menyimpan properti `DateSent` dan `DateReceived`, dan keduanya ditulis ke format target secara otomatis.
