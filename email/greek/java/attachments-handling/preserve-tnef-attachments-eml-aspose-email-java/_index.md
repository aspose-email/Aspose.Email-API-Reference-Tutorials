## Οδηγός Υλοποίησης

### Φόρτωση EML με τη λειτουργία PreserveTnefAttachments

#### Επισκόπηση

Η σημαία `PreserveTnefAttachments` λέει στο Maven Aspose Email να διατηρήσει τα αρχικά αρχεία TNEF ακριβώς όπως εμφανίζονται στο πηγαίο EML, αποτρέποντας οποιαδήποτε αυτόματη μετατροπή.

#### Υλοποίηση Βήμα‑βήμα

**1. Διαμόρφωση Επιλογών Φόρτωσης**

Δημιουργήστε ένα αντικείμενο `EmlLoadOptions` και ενεργοποιήστε τη διατήρηση TNEF:

```java
EmlLoadOptions options = new EmlLoadOptions();
options.setPreserveTnefAttachments(true);
```

**2. Φόρτωση του Αρχείου EML**

Περάστε τις επιλογές κατά τη φόρτωση του μηνύματος:

```java
MailMessage eml = MailMessage.load(dataDir + "tnefEml.eml", options);
```

**3. Πρόσβαση στα Συνημμένα**

Περιηγηθείτε στα συνημμένα για να επαληθεύσετε ότι παραμένουν αρχεία TNEF:

```java
for (Attachment attachment : eml.getAttachments()) {
    System.out.println(attachment.getName());
}
```

#### Συμβουλές Επίλυσης Προβλημάτων

- **Λείπουν Συνημμένα** – ελέγξτε ξανά τη διαδρομή αρχείου (`dataDir`) και βεβαιωθείτε ότι το αρχείο είναι αναγνώσιμο.  
- **Απροσδόκητη Μετατροπή** – βεβαιωθείτε ότι καλείται `setPreserveTnefAttachments(true)` πριν από τη φόρτωση του μηνύματος.

Κατά την επεξεργασία μεγάλων παρτίδων email με το Maven Aspose Email:
