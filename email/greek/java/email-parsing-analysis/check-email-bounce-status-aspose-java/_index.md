## Οδηγός Εφαρμογής

Αυτή η ενότητα καλύπτει τις λειτουργίες για τον έλεγχο της κατάστασης επιστροφής μηνυμάτων email χρησιμοποιώντας το Aspose.Email.

### Φόρτωση και έλεγχος κατάστασης αναπήδησης ενός μεμονωμένου μηνύματος ηλεκτρονικού ταχυδρομείου

#### Επισκόπηση
Αυτή η λειτουργία επιδεικνύει τη φόρτωση ενός μεμονωμένου αρχείου email για να διαπιστωθεί εάν έχει αποσυρθεί, λαμβάνοντας βασικές λεπτομέρειες σχετικά με την αποκάλυψη.

#### Βήματα Υλοποίησης
**Βήμα 1: Εισαγωγή απαιτούμενων βιβλιοθηκών**
Ξεκινήστε εισάγοντας τις απαραίτητες κλάσεις:

```java
import com.aspose.email.BounceResult;
import com.aspose.email.MailMessage;
```

**Βήμα 2: Φόρτωση αρχείου μηνύματος ηλεκτρονικού ταχυδρομείου**
Καθορίστε τον κατάλογο και το όνομα αρχείου για το μήνυμα email σας και, στη συνέχεια, φορτώστε το χρησιμοποιώντας `MailMessage.load()`.

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY/";
String fileName = "failed.msg";
MailMessage mail = MailMessage.load(dataDir + fileName);
```

**Βήμα 3: Έλεγχος κατάστασης εγκατάλειψης**
Χρησιμοποιήστε το `checkBounced()` μέθοδος για να διαπιστώσετε εάν το email έχει αποτύχει και να ανακτήσετε βασικές λεπτομέρειες για το απόρριψη:

```java
BounceResult result = mail.checkBounced();
```

**Βήμα 4: Πρόσβαση στις Ιδιότητες Bounce**
Πρόσβαση σε ιδιότητες όπως η κατάσταση εγκατάλειψης, η ενέργεια που πραγματοποιήθηκε λόγω της εγκατάλειψης και οι πληροφορίες του παραλήπτη:

```java
System.out.println("IsBounced : " + result.isBounced());
System.out.println("Action : " + result.getAction());
System.out.println("Recipient : " + result.getRecipient());
```

### Φόρτωση και έλεγχος λεπτομερούς κατάστασης αναπήδησης ενός μηνύματος ηλεκτρονικού ταχυδρομείου

#### Επισκόπηση
Αυτή η λειτουργία επεκτείνει την πρώτη, ανακτώντας λεπτομερείς πληροφορίες σχετικά με το γιατί το email αναπήδησε.

#### Βήματα Υλοποίησης
Ακολουθήστε παρόμοια βήματα όπως πριν, αλλά αποκτήστε πρόσβαση σε περισσότερες ιδιότητες για αναλυτικές λεπτομέρειες:
**Βήμα 1 έως Βήμα 3:** Το ίδιο όπως στο Χαρακτηριστικό 1.

**Βήμα 4: Πρόσβαση σε λεπτομερείς ιδιότητες εγκατάλειψης**
Εκτός από τις βασικές ιδιότητες, λάβετε λεπτομερείς λόγους εγκατάλειψης και κατάσταση:

```java
System.out.println("Reason : " + result.getReason());
System.out.println("Status : " + result.getStatus());
System.out.println("OriginalMessage ToAddress 1: " + 
    result.getOriginalMessage().getTo().get_Item(0).getAddress());
```

### Φόρτωση και έλεγχος κατάστασης αναπήδησης ενός άλλου μηνύματος ηλεκτρονικού ταχυδρομείου

#### Επισκόπηση
Το τρίτο χαρακτηριστικό απεικονίζει τη διαδικασία για ένα διαφορετικό αρχείο email, δίνοντας έμφαση στην επαναχρησιμοποίηση.

**Βήματα Υλοποίησης:** Ακολουθήστε τα βήματα παρόμοια με αυτά της Λειτουργίας 1, προσαρμόζοντας το όνομα αρχείου όπως απαιτείται:

```java
String fileName = "test.eml";
MailMessage mail = MailMessage.load(dataDir + fileName);
BounceResult result = mail.checkBounced();
// Αποκτήστε πρόσβαση στις ιδιότητες με παρόμοιο τρόπο.
```
