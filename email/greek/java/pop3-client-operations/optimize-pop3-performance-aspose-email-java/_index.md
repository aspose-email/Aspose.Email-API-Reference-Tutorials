Ξεκινήστε αρχικοποιώντας το `Pop3Client`:

## Οδηγός Εφαρμογής
### Διαμόρφωση λειτουργίας πολλαπλών συνδέσεων
**Επισκόπηση:**  
Η λειτουργία πολλαπλών συνδέσεων χρησιμοποιεί πολλαπλές ταυτόχρονες συνδέσεις σε έναν διακομιστή POP3, βελτιώνοντας την ταχύτητα και την απόδοση ανάκτησης.

#### Ρύθμιση πολλαπλών συνδέσεων
1. **Ενεργοποίηση λειτουργίας πολλαπλής σύνδεσης:**
   
   ```java
   import com.aspose.email.Pop3MessageInfoCollection;
   
pop3Client.setUseMultiConnection(MultiConnectionMode.Enable);
   ```

2. **Configure Connections Quantity:**
   
   ```java
   pop3Client.setConnectionsQuantity(5); // Use 5 connections for improved performance
   ```

3. **Λίστα μηνυμάτων χρησιμοποιώντας πολλαπλές συνδέσεις:**
   
   ```java
   long multiConnectionModeStartTime = System.nanoTime();
   Pop3MessageInfoCollection messageInfoCol1 = pop3Client.listMessages(); 
   long multiConnectionModeTimeSpan = System.nanoTime() - multiConnectionModeStartTime;
   ```

### Διαμόρφωση λειτουργίας μίας σύνδεσης
**Επισκόπηση:**  
Η λειτουργία μίας σύνδεσης είναι ο παραδοσιακός τρόπος αλληλεπίδρασης με έναν διακομιστή POP3, χρήσιμος για περιβάλλοντα όπου οι συνδέσεις είναι περιορισμένες.

#### Ρύθμιση μίας σύνδεσης
1. **Απενεργοποίηση πολλαπλών συνδέσεων:**
   
   ```java
   pop3Client.setUseMultiConnection(MultiConnectionMode.Disable);
   ```

2. **Λίστα μηνυμάτων χρησιμοποιώντας μία μόνο σύνδεση:**
   
   ```java
   long singleConnectionModeStartTime = System.nanoTime();
   Pop3MessageInfoCollection messageInfoCol2 = pop3Client.listMessages(); 
   long singleConnectionModeTimeSpan = System.nanoTime() - singleConnectionModeStartTime;
   ```

### Σύγκριση απόδοσης
**Επισκόπηση:**  
Η κατανόηση της επίδρασης κάθε λειτουργίας στην απόδοση βοηθά στην επιλογή της σωστής προσέγγισης.

1. **Υπολογισμός του λόγου απόδοσης:**
   
   ```java
   double performanceRelation = (double)singleConnectionModeTimeSpan / (double)multiConnectionModeTimeSpan;
   System.out.println("Performance Relation: " + performanceRelation);
   ```

   Αυτός ο υπολογισμός υποδεικνύει πόσο ταχύτερη είναι η λειτουργία πολλαπλών συνδέσεων σε σύγκριση με τη λειτουργία μίας σύνδεσης.
