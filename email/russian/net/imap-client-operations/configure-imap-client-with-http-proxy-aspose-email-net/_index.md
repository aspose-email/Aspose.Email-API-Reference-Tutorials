## Руководство по внедрению

### Настройка IMAP-клиента с HTTP-прокси

#### Обзор
Эта функция позволяет настроить HTTP-прокси для доступа к электронной почте по протоколу IMAP, что особенно важно, когда сетевые политики требуют, чтобы весь трафик проходил через определенный сервер.

**Шаг 1: Создание экземпляра HttpProxy**
```csharp
// Инициализируйте HttpProxy, указав адрес хоста и номер порта.
HttpProxy proxy = new HttpProxy("18.222.124.59", 8080);
```
- **Параметры**: IP или имя хоста (`"18.222.124.59"`) и номер порта (`8080`).

**Шаг 2: Инициализация ImapClient**
```csharp
using (ImapClient client = new ImapClient("imap.domain.com", "username", "password"))
{
    // Назначьте прокси-сервер свойству Proxy клиента.
    client.Proxy = proxy;

    // Выберите папку «Входящие».
    client.SelectFolder("Inbox");
}
```
- **Цель**: `ImapClient` подключает вас к вашему почтовому серверу. Использование прокси-сервера гарантирует правильную маршрутизацию всех запросов.

**Совет по устранению неполадок**: Для успешного подключения убедитесь, что настройки прокси-сервера соответствуют настройкам, предоставленным администратором сети.

### Базовая инициализация IMAP-клиента и выбор папки

#### Обзор
Для сред без HTTP-прокси эта функция позволяет выполнить базовую инициализацию клиента IMAP для прямого доступа к папкам электронной почты, таким как «Входящие».

**Шаг 1: Инициализация ImapClient**
```csharp
using (ImapClient client = new ImapClient("imap.domain.com", "username", "password"))
{
    // Выберите папку, с которой вы хотите работать.
    client.SelectFolder("Inbox");
}
```
- **Объяснение**: Этот шаг подключается к вашему почтовому серверу без прокси. Убедитесь, что используются правильные учетные данные.
