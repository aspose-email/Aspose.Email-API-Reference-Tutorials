## Базовая инициализация и настройка

## Руководство по внедрению

### Инициализация клиента Exchange
Создайте подключение к серверу Exchange:

#### Обзор
Создание экземпляра `ExchangeClient` позволяет взаимодействовать с вашим сервером Exchange. Этот шаг включает предоставление необходимых учетных данных и URL-адресов конечных точек.

#### Шаги
1. **Включить требуемые пространства имен**:
   ```csharp
   using Aspose.Email.Clients.Exchange.WebService;
   ```
2. **Инициализация клиента**:
   ```csharp
   IEWSClient client = EWSClient.GetEWSClient(
       "https://outlook.office365.com/ews/exchange.asmx", 
       "testUser", 
       "pwd", 
       "domain"
   );
   ```
   - `GetEWSClient`: Подключается к серверу Exchange, используя предоставленные учетные данные.
   - Параметры:
     - URL конечной точки: адрес конечной точки ваших веб-служб Exchange.
     - Имя пользователя, пароль, домен: учетные данные для аутентификации.

### Извлечение задач из Exchange Server

#### Обзор
Извлечение задач позволяет расставлять приоритеты и управлять рабочей нагрузкой.

#### Шаги
1. **Доступ к URI задачи**:
   ```csharp
   using Aspose.Email.Clients.Exchange.WebService;

   public static void ListExchangeTasks(IEWSClient client)
   {
       ExchangeMessageInfoCollection tasks = client.ListMessages(client.MailboxInfo.TasksUri);
   }
   ```
   - `ListMessages`: Извлекает все сообщения, связанные с задачей, с сервера.

### Фильтрация и удаление задач по теме

#### Обзор
Фильтрация и удаление определенных задач поддерживает чистоту рабочего пространства, гарантируя, что активными останутся только соответствующие задачи.

#### Шаги
1. **Итерация по коллекции задач**:
   ```csharp
   using Aspose.Email.Clients.Exchange.WebService;
   using Aspose.Email.Mime;

   public static void FilterAndDeleteTasks(IEWSClient client)
   {
       foreach (ExchangeMessageInfo info in client.ListMessages(client.MailboxInfo.TasksUri))
       {
           ExchangeTask task = client.FetchTask(info.UniqueUri);
           
           if (task.Subject.Equals("test"))
           {
               client.DeleteItem(task.UniqueUri, DeletionOptions.DeletePermanently);
           }
       }
   }
   ```
   - `FetchTask`: Извлекает подробную информацию о конкретной задаче, используя ее уникальный URI.
   - `DeleteItem`: Удаляет задачу навсегда с сервера.

### Советы по устранению неполадок
- **Ошибки аутентификации**: Проверьте учетные данные и URL конечной точки. Проверьте наличие сетевых проблем, препятствующих доступу.
- **Проблемы с разрешением**: Убедитесь, что у учетной записи пользователя есть разрешения на перечисление и удаление задач на сервере Exchange.
