## Руководство по внедрению

### Функция 1: Обучение и сохранение базы данных спам-фильтров

В этом разделе вы узнаете, как обучить байесовский спам-фильтр с использованием как не спамовых, так и некачественных писем, а затем сохраните обученную базу данных.

#### Обзор

Основная идея здесь заключается в анализе образцов электронной почты — различении легитимных и спам-сообщений — для обучения вашего фильтра. После того, как модель будет должным образом обучена, ее можно сохранить для будущего использования.

#### Шаги по реализации

**1. Определите пути к файлам**
Начните с настройки путей для папок с нежелательной почтой и спамом, а также для выходного файла базы данных:

```csharp
string hamFolder = "YOUR_DOCUMENT_DIRECTORY/hamFolder";
string spamFolder = "YOUR_DOCUMENT_DIRECTORY/spamFolder";
string dataBaseFile = "YOUR_OUTPUT_DIRECTORY/SpamFilterDatabase.txt";
```

**2. Загрузка файлов электронной почты**
Получить все `.eml` файлы из этих каталогов для использования в обучении:

```csharp
string[] hamFiles = Directory.GetFiles(hamFolder, "*.eml");
string[] spamFiles = Directory.GetFiles(spamFolder, "*.eml");
```

**3. Инициализируйте SpamAnalyzer**
Создайте новый экземпляр `SpamAnalyzer`, который будет использоваться как для обучения, так и для тестирования.

```csharp
SpamAnalyzer analyzer = new SpamAnalyzer();
```

**4. Обучите фильтр с помощью некачественных писем**
Просмотрите все нежелательные письма, чтобы обучить свой фильтр, отмечая каждое из них как не спам:

```csharp
foreach (string file in hamFiles)
{
    try
    {
        MailMessage hamMailMessage = MailMessage.Load(file);
        analyzer.TrainFilter(hamMailMessage, false);
    }
    catch (Exception) 
    {
        continue; // Пропустить файлы, которые невозможно загрузить
    }
}
```

**5. Обучите фильтр спам-письмам**
Аналогичным образом просмотрите спам-письма, чтобы пометить их как спам:

```csharp
foreach (string file in spamFiles)
{
    try
    {
        MailMessage spamMailMessage = MailMessage.Load(file);
        analyzer.TrainFilter(spamMailMessage, true);
    }
    catch (Exception) 
    {
        continue; // Пропустить файлы, которые невозможно загрузить
    }
}
```

**6. Сохраните обученную базу данных**
После завершения обучения сохраните модель в файл:

```csharp
analyzer.SaveDatabase(dataBaseFile);
```

### Функция 2: Тестирование писем с помощью спам-фильтра

После обучения и сохранения базы данных спам-фильтра вы можете проверять новые электронные письма на предмет вероятности попадания в спам.

#### Обзор

Эта функция демонстрирует загрузку обученной базы данных и ее применение для классификации новых писем как нежелательных или спамовых на основе оценки вероятности.

#### Шаги по реализации

**1. Загрузка обученной базы данных**
Инициализировать `SpamAnalyzer` с путем к сохраненной базе данных:

```csharp
string dataBaseFile = "YOUR_OUTPUT_DIRECTORY/SpamFilterDatabase.txt";
SpamAnalyzer analyzer = new SpamAnalyzer(dataBaseFile);
```

**2. Извлечение и проверка электронных писем**
Загрузите электронные письма из тестового каталога, а затем используйте обученный фильтр для их оценки:

```csharp
string[] testFiles = Directory.GetFiles("YOUR_DOCUMENT_DIRECTORY", "*.eml");

foreach (string file in testFiles)
{
    MailMessage msg = MailMessage.Load(file);
    double probability = analyzer.Test(msg);

    // Вывод результатов на основе вероятности
    PrintResult(probability);
}

void PrintResult(double probability)
{
    if (probability < 0.05) Console.WriteLine("This is ham");
    else if (probability > 0.95) Console.WriteLine("This is spam");
    else Console.WriteLine("Maybe spam");
}
```
