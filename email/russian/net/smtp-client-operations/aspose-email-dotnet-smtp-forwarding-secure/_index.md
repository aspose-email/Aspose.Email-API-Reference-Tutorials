### Пересылка электронной почты SMTP

Пересылка электронной почты SMTP позволяет автоматически отправлять электронные письма от одного получателя другому, что необходимо для маршрутизации сообщений в автоматизированном рабочем процессе.

#### Шаг 1: Определите SMTP-клиент с данными сервера
Настройте свой `SmtpClient` пример:
```csharp
private static void InitializeSmtpClient(SmtpClient client)
{
    // Задайте хост, имя пользователя, пароль, номер порта и параметры безопасности для SMTP-подключения.
    client.Host = "mail.server.com";
    client.Username = "username";
    client.Password = "password";
    client.Port = 587;
    client.SecurityOptions = SecurityOptions.SSLExplicit; // Используйте SSL явно
}
```
#### Шаг 2: Загрузите электронное письмо
Загрузите сообщение электронной почты, которое вы хотите переслать:
```csharp
string dataDir = "/YOUR_DOCUMENT_DIRECTORY/Message.eml";
MailMessage message = MailMessage.Load(dataDir);
```
#### Шаг 3: Перешлите электронное письмо
Используйте `Forward` способ отправки указанным получателям:
```csharp
client.Forward("Recipient1@domain.com", "Recipient2@domain.com", message);
```
### Безопасная конфигурация электронной почты

Обеспечение безопасной электронной переписки имеет первостепенное значение. Библиотека Aspose.Email упрощает настройку параметров безопасности для SMTP.

#### Шаг 1: Инициализация и настройка SmtpClient с параметрами безопасности
Настройте `SmtpClient` с явным SSL:
```csharp
private static void SetupSecurityOptions(SmtpClient client)
{
    // Установить данные хоста для SMTP-сервера
    client.Host = "mail.server.com";
    client.Username = "username";
    client.Password = "password";
    client.Port = 587; // Общий порт для безопасных соединений

    // Используйте SSL явно, чтобы гарантировать шифрование связи.
    client.SecurityOptions = SecurityOptions.SSLExplicit;
}
```

## Раздел часто задаваемых вопросов

Вот некоторые распространенные вопросы и ответы, связанные с использованием Aspose.Email для .NET:
1. **Как обрабатывать исключения во время операций SMTP?**
   - Используйте блоки try-catch `SmtpClient` методы изящного управления ошибками, связанными с сетью.
2. **Что делать, если мой SMTP-сервер использует другой порт?**
   - Отрегулируйте `Port` недвижимость в вашем `SmtpClient` конфигурация соответственно.
3. **Могу ли я пересылать электронные письма с вложениями?**
   - Да, загрузите электронное письмо с вложениями и используйте тот же метод пересылки, который показан выше.
4. **Как защитить свои учетные данные SMTP?**

5. **Какие существуют альтернативы SSLExplicit с точки зрения безопасности?**
