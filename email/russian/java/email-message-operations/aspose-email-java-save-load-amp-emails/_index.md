## Руководство по внедрению

### Сохранение электронного письма с помощью компонентов AMP
**Обзор**: эта функция позволяет сохранить электронное письмо, гарантируя правильное сохранение всех компонентов AMP.

#### Шаг 1: Загрузите электронное письмо
Сначала загрузите существующее сообщение электронной почты:
```java
import com.aspose.email.MailMessage;
import com.aspose.email.AmpMessage;

String dataDir = "YOUR_DOCUMENT_DIRECTORY/OutputDirectory/";
MailMessage savedMsg = MailMessage.load(dataDir + "AmpTest_1.eml");
```

#### Шаг 2: Проверка и добавление компонента AMP
Убедитесь, что электронная почта является `AmpMessage` экземпляр перед добавлением компонентов:
```java
if (savedMsg instanceof AmpMessage) {
    import com.aspose.email.AmpTimeago;
    import java.util.Date;

    Date dt = new Date();
    
    // Добавить компонент AmpTimeago
    AmpTimeago time = new AmpTimeago(dt);
    time.getAttributes().setWidth(600);
    time.getAttributes().setHeight(300);
    time.getAttributes().setLayout(LayoutType.Fixed);
    time.setLocale("en-US");
    time.setCutoff(600);

    ((AmpMessage)savedMsg).addAmpComponent(time);
}
```

#### Шаг 3: Сохраните обновленное электронное письмо.
Наконец, сохраните электронное письмо с недавно добавленным компонентом AMP:
```java
((AmpMessage)savedMsg).save(dataDir + "AmpTest_2.eml");
```

### Советы по устранению неполадок
- **Отсутствующие зависимости**: Убедитесь, что все необходимые зависимости правильно объявлены в вашем `pom.xml`.
- **Неправильный путь**: Еще раз проверьте пути к файлам, чтобы убедиться, что они указывают на правильные каталоги.
- **Ошибки компонентов AMP**: Убедитесь, что добавляемые вами компоненты AMP совместимы с существующей структурой электронного письма.
