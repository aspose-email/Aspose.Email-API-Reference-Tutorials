## Руководство по реализации

Разделим реализацию на три чётко определённые функции.

### Функция 1: Загрузка MailMessage из файла

#### Обзор
Загрузка файла `.msg` предоставляет полный программный доступ к содержимому письма, вложениям и метаданным.

#### Пошагово

**Импортировать необходимые классы**

```java
import com.aspose.email.MailMessage;
import com.aspose.email.MsgLoadOptions;
```

**Загрузить письмо**

```java
String filename = "YOUR_DOCUMENT_DIRECTORY/MSG file with RTF Formatting.msg";
MailMessage msg = MailMessage.load(filename, new MsgLoadOptions());
```

`MsgLoadOptions` позволяет управлять тем, как интерпретируется MSG‑файл; настройки по умолчанию подходят для большинства сценариев.

### Функция 2: Установка текущей даты и пользовательского смещения часового пояса

#### Обзор
Точные метки времени критичны, когда вы работаете с пользователями из разных регионов.

**Установить текущую дату**

```java
import java.util.Date;

msg.setDate(new Date());
```

**Применить пользовательское смещение часового пояса (например, UTC+5)**

```java
msg.setTimeZoneOffset(5 * 60 * 60 * 1000); // 5 hours ahead of UTC in milliseconds.
```

Смещение задаётся в миллисекундах, поэтому для зон к западу от UTC можно передать отрицательные значения.

### Функция 3: Сохранение MailMessage как файла MHTML

#### Обзор
MHTML объединяет HTML‑контент и встроенные ресурсы в один файл, что идеально подходит для архивирования или обмена.

**Настроить параметры сохранения**

```java
import com.aspose.email.MhtSaveOptions;
import com.aspose.email.MhtFormatOptions;

MhtSaveOptions mhtOptions = new MhtSaveOptions();
mhtOptions.setMhtFormatOptions(MhtFormatOptions.WriteHeader);
```

**Сохранить письмо**

```java
msg.save("YOUR_OUTPUT_DIRECTORY/ExportToMHTWithCustomTimezone_out.mhtml", mhtOptions);
```

Полученный файл `.mhtml` сохраняет оригинальное форматирование, изображения и вложения.

Конвертация MSG‑файлов в MHTML даёт веб‑дружественное представление в виде одного файла, которое можно открыть в любом современном браузере. Это особенно полезно для:

- **Юридического архивирования**, где требуется точная визуальная копия.
- **Кросс‑платформенного обмена** без необходимости в Outlook.
- **Встраивания писем** в веб‑страницы или документацию.

## Часто задаваемые вопросы

**В: Можно ли загружать письма из форматов, отличных от .msg?**  
О: Да, Aspose.Email поддерживает EML, MSG, MHT и несколько других форматов.

**В: Как эффективно обрабатывать очень большие файлы писем?**  
О: Используйте потоковые API Aspose.Email для чтения/записи данных порциями, уменьшая нагрузку на память.

**В: Можно ли изменять вложения внутри MailMessage?**  
О: Абсолютно. Вы можете добавлять, удалять или заменять вложения через коллекцию `MailMessage.getAttachments()`.

**В: Что делать, если смещение часового пояса отрицательное (западнее UTC)?**  
О: Передайте отрицательное значение в миллисекундах в `setTimeZoneOffset`, например `-3 * 60 * 60 * 1000` для UTC‑3.

**В: Можно ли использовать Aspose.Email в коммерческих проектах?**  
О: Да, при наличии действующей коммерческой лицензии.

**В: Как обрабатывать тысячи MSG‑файлов, не исчерпав память?**  
О: Обрабатывайте файлы пакетами, освобождайте каждый `MailMessage` после сохранения и используйте конструкцию `try‑with‑resources` Java для автоматической очистки.
