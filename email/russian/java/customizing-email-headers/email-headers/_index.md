# Как читать заголовки и создавать пользовательские заголовки электронной почты с Aspose.Email

## Введение в заголовки электронной почты

В этом руководстве вы узнаете **как читать заголовки**, научитесь **как добавлять значения заголовков** и поймёте, почему пользовательские заголовки электронной почты важны для современных рабочих процессов обмена сообщениями. Заголовки письма работают как цифровой конверт, содержащий метаданные: отправитель, получатель, путь маршрутизации и метки времени. К концу этого руководства вы сможете **извлекать заголовки письма**, создавать собственные пользовательские поля и читать заголовок темы письма — всё с помощью Aspose.Email для Java.

## Быстрые ответы
- **Как основной способ добавить пользовательский заголовок?** Используйте коллекцию `Headers` объекта `MailMessage`.  
- **Как прочитать заголовок Subject после загрузки письма?** Вызовите `message.getHeaders().get("Subject")`.  
- **Нужна ли лицензия для использования API заголовков?** Пробная версия подходит для разработки; для продакшн‑использования требуется коммерческая лицензия.  
- **Есть ли ограничения на имена пользовательских заголовков?** Следуйте правилам именования RFC 5322 (например, начинайте с «X-»).  
- **Какая версия Aspose.Email поддерживает эти возможности?** Все последние версии (2024‑2026) включают полное управление заголовками.

## Что такое заголовок и зачем его читать?

Заголовки — это строки обычного текста, размещённые в верхней части сообщения электронной почты. Они описывают, кто отправил сообщение, когда оно было отправлено и как оно проходило через почтовые серверы. Возможность **читать заголовки** позволяет вам:

* Диагностировать проблемы доставки, проверяя цепочку `Received`.  
* Сопоставлять сообщения с внутренними идентификаторами задач (`X-Job-ID`).  
* Реализовывать пользовательскую логику маршрутизации, используя поля вроде `X-Priority`.

## Как добавить пользовательский заголовок (Создание пользовательских заголовков электронной почты)

### Шаг 1: Инициализировать MailMessage

```java
MailMessage message = new MailMessage();
```

### Шаг 2: Добавить пользовательский заголовок

```java
message.getHeaders().add("X-Custom-Header", "My Custom Value");
```

> **Подсказка:** Добавляйте префикс `X-` к пользовательским заголовкам, чтобы соответствовать RFC 5322 и избегать конфликтов со стандартными полями.

### Шаг 3: Отправить письмо

```java
SmtpClient client = new SmtpClient("smtp.example.com");
client.send(message);
```

## Как читать заголовки электронной почты (Чтение заголовка темы письма)

### Шаг 1: Загрузить письмо из файла или потока

```java
MailMessage message = MailMessage.load("path/to/email.eml");
```

### Шаг 2: Извлечь любой нужный заголовок

```java
String subject = message.getHeaders().get("Subject");
String sender = message.getHeaders().get("From");
```

> **Примечание:** Коллекция `Headers` возвращает `null`, если запрашиваемый заголовок отсутствует, поэтому всегда проверяйте значение на `null` перед использованием.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|----------|
| Заголовок не появляется в полученном письме | SMTP‑сервер удаляет неизвестные заголовки | Убедитесь, что сервер позволяет пользовательские заголовки `X-` или настройте его на их сохранение. |
| `null` возвращается при чтении заголовка | Опечатка в имени заголовка (чувствительность к регистру) | Используйте точное имя заголовка, как хранится, например, `"Subject"`, а не `"subject"`. |
| Дублирующиеся заголовки | Добавление одного и того же заголовка несколько раз | Используйте `addOrUpdate` (если доступно) или удалите старую запись перед добавлением новой. |

## Часто задаваемые вопросы

**В: Как просмотреть заголовки письма в популярных почтовых клиентах?**  
О: Большинство клиентов позволяют просмотреть исходный код письма — ищите опции «View Original», «Show Headers» или «View Source».

**В: Шифруются ли заголовки письма?**  
О: Нет. Заголовки — это открытые текстовые метаданные и передаются в открытом виде, если только всё сообщение не зашифровано (например, S/MIME).

**В: Можно ли изменить заголовки письма после его отправки?**  
О: После того как сообщение отправлено, заголовки становятся неизменяемыми. Установите все необходимые заголовки **до** вызова `client.send(message)`.

**В: Какова цель заголовка “Received”?**  
О: Он фиксирует каждый переход письма, помогая администраторам диагностировать проблемы доставки и отслеживать путь сообщения.

**В: Как извлечь заголовки писем из большой партии писем?**  
О: Используйте `MailMessage.load` Aspose.Email в цикле или примените `MailMessageCollection` для пакетной обработки.
