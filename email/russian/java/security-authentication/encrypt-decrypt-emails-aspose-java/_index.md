## Руководство по внедрению

### Функция 1: Шифрование сообщения

Шифрование вашего электронного письма гарантирует, что прочитать его сможет только тот получатель, у которого есть соответствующий закрытый ключ.

#### Обзор
Мы покажем, как использовать Aspose.Email для Java для шифрования электронной почты с использованием открытого сертификата (.cer).

#### Пошаговый процесс

##### **Настройка путей к файлам и импорт библиотек**

Начните с указания каталога документов и импорта необходимых классов:

```java
import com.aspose.email.MailMessage;
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;

String dataDir = "YOUR_DOCUMENT_DIRECTORY";
String publicCertFileName = dataDir + "/MyKey.cer";
Path publicCertFilePath = Paths.get(publicCertFileName);
```

##### **Создать и зашифровать сообщение**

Создать `MailMessage` объект, затем зашифруйте его с помощью открытого сертификата:

```java
// Создать сообщение
MailMessage msg = new MailMessage("sender@example.com", "receiver@example.com",
                                  "Test subject", "Test Body");

// Зашифровать сообщение
MailMessage eMsg = null;
try {
    // Прочитайте публичный сертификат и зашифруйте сообщение.
    eMsg = msg.encrypt(Files.readAllBytes(publicCertFilePath), "");
} catch (IOException e) {
    e.printStackTrace();
}
```

#### Ключевые соображения
- Убедитесь, что ваш `.cer` Путь к файлу правильный.
- Обрабатывайте исключения, чтобы избежать сбоев программы во время шифрования.

### Функция 2: Проверка статуса шифрования сообщения

После шифрования проверьте статус сообщения, чтобы убедиться, что оно было успешно зашифровано.

```java
// Проверьте, зашифровано ли сообщение электронной почты.
if (eMsg != null && eMsg.isEncrypted()) {
    System.out.println("The message is encrypted.");
} else if (eMsg != null) {
    System.out.println("The message is not encrypted.");
}
```

### Функция 3: Расшифровка сообщения

Расшифровка электронного письма позволяет вам получить безопасный доступ к его содержимому, гарантируя, что просматривать его смогут только авторизованные пользователи с правильным закрытым ключом.

#### Обзор
Теперь мы расшифруем ранее зашифрованное сообщение, используя закрытый сертификат (.pfx).

#### Пошаговый процесс

##### **Настройка путей к файлам и импорт библиотек**

Убедитесь, что указан путь к вашему личному сертификату:

```java
String dataDir = "YOUR_DOCUMENT_DIRECTORY";
String privateCertFileName = dataDir + "/MyPFX.pfx";
Path privateCertFilePath = Paths.get(privateCertFileName);
```

##### **Расшифровать сообщение**

Используйте вспомогательный метод для расшифровки сообщения электронной почты:

```java
// Расшифровать зашифрованное сообщение
decryptMessage(eMsg, privateCertFilePath, "password");

// Вспомогательный метод для расшифровки сообщения
void decryptMessage(MailMessage eMsg, Path privateCertFilePath, String password) {
    if (eMsg == null) return;

    MailMessage dMsg = null;
    try {
        // Прочитайте закрытый сертификат и расшифруйте сообщение.
        dMsg = eMsg.decrypt(Files.readAllBytes(privateCertFilePath), password);
        
        // Проверить статус расшифровки
        if (dMsg != null && !dMsg.isEncrypted()) {
            System.out.println("The message has been successfully decrypted.");
        }
    } catch (IOException ex) {
        ex.printStackTrace();
    }
}
```

#### Ключевые соображения
- Проверьте путь и пароль вашего `.pfx` файл.
- Используйте обработку исключений для корректного управления ошибками дешифрования.

### Функция 4: Проверка статуса шифрования расшифрованного сообщения

Убедитесь, что расшифрованное сообщение больше не зашифровано:

```java
// Убедитесь, что сообщение не зашифровано после расшифровки.
if (dMsg != null && !dMsg.isEncrypted()) {
    System.out.println("The message has been successfully decrypted.");
} else if (dMsg != null) {
    System.out.println("Failed to decrypt the message properly.");
}
```
