## कार्यान्वयन मार्गदर्शिका

### EWS क्लाइंट का उपयोग करके Exchange सर्वर से कनेक्ट करें

#### अवलोकन
ईमेल डेटा तक प्रोग्रामेटिक रूप से पहुंचने और उसे प्रबंधित करने के लिए एक्सचेंज सर्वर के साथ कनेक्शन स्थापित करना महत्वपूर्ण है।

#### चरण-दर-चरण मार्गदर्शिका
**1. क्रेडेंशियल परिभाषित करें**
उपयोगकर्ता नाम, पासवर्ड और डोमेन (यदि लागू हो) सहित अपने नेटवर्क क्रेडेंशियल सेट करें।
```csharp
const string mailboxUri = "https://एक्सचेंज/ईडब्ल्यूएस/एक्सचेंज.एएसएमएक्स";
const string domain = @"";
const string username = "username@ASE305.onmicrosoft.com";
const string password = "password";
NetworkCredential credentials = new NetworkCredential(username, password, domain);
```

**2. संपर्क स्थापित करें**
उपयोग `EWSClient.GetEWSClient` अपने Exchange सर्वर से कनेक्ट करने की विधि.
```csharp
IEWSClient client = EWSClient.GetEWSClient(mailboxUri, credentials);
```

### हटाए गए आइटम फ़ोल्डर से संदेशों की सूची बनाएं

#### अवलोकन
विशिष्ट फ़ोल्डरों से संदेश प्राप्त करने से ईमेल डेटा को कुशलतापूर्वक प्रबंधित और विश्लेषण करने में मदद मिलती है। हम इस ट्यूटोरियल के लिए हटाए गए आइटम फ़ोल्डर पर ध्यान केंद्रित करेंगे।

**3. ईमेल पुनः प्राप्त करें**
एक बार कनेक्ट होने के बाद, का उपयोग करें `ListMessages` हटाए गए आइटम फ़ोल्डर में ईमेल तक पहुंचने की विधि।
```csharp
using Aspose.Email.Clients.Exchange;
using System.Collections.Generic;

ExchangeMessageInfoCollection list = client.ListMessages(client.MailboxInfo.DeletedItemsUri);

// संदेश प्रकारों को दोहराएँ और प्रदर्शित करें.
foreach (var messageInfo in list)
{
    Console.WriteLine(messageInfo.MessageInfoType.ToString());
}
```

#### स्पष्टीकरण
- **`ListMessages`**: निर्दिष्ट फ़ोल्डर URI से संदेशों का संग्रह पुनर्प्राप्त करता है।
- **संदेश जानकारी प्रकार**: प्रत्येक संदेश के बारे में जानकारी प्रदान करता है, जैसे कि यह ईमेल है या कैलेंडर आइटम।

### समस्या निवारण युक्तियों

- एक्सचेंज सर्वर के साथ कनेक्शन संबंधी समस्याओं से बचने के लिए नेटवर्क कनेक्टिविटी की जांच करें।
- सत्यापित करें कि Aspose.Email आपके प्रोजेक्ट में सही ढंग से स्थापित और संदर्भित है।
