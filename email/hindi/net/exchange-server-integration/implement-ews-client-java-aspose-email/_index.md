## कार्यान्वयन मार्गदर्शिका

### विशेषता 1: EWS क्लाइंट आरंभीकरण

**अवलोकन:** EWS क्लाइंट को आरंभ करना जावा अनुप्रयोगों के माध्यम से एक्सचेंज सर्वर कार्यात्मकताओं तक पहुंचने की दिशा में आपका पहला कदम है।

#### चरण-दर-चरण प्रक्रिया:

**3.1 आवश्यक पैकेज आयात करें**

सुनिश्चित करें कि आप Aspose.Email और नेटवर्किंग क्षमताओं के लिए आवश्यक पैकेज आयात करें।

```java
import com.aspose.email.IEWSClient;
import com.aspose.email.EWSClient;
```

**3.2 क्लाइंट को प्रारंभ करें**

सेवा URL, उपयोगकर्ता नाम, पासवर्ड और डोमेन सहित मान्य क्रेडेंशियल्स का उपयोग करके अपना क्लाइंट सेट अप करें।

```java
public class EWSServiceInitialization {
    public static void main(String[] args) {
        try {
            IEWSClient client = EWSClient.getEWSClient(
                "https://आउटलुक.ऑफिस365.com/ews/एक्सचेंज.asmx",
                "testUser",
                "pwd",
                "domain"
            );
        } catch (Exception ex) {
            System.out.println(ex.getMessage());
        }
    }
}
```

**स्पष्टीकरण:**
- The `getEWSClient` विधि कनेक्शन को प्रमाणित करने और स्थापित करने के लिए सेवा URL, उपयोगकर्ता नाम, पासवर्ड और डोमेन के लिए पैरामीटर लेती है।
- कनेक्टिविटी समस्याओं के प्रबंधन के लिए हमेशा अपवादों को शालीनता से संभालें।

### फ़ीचर 2: इनबॉक्स से संदेशों की सूची बनाएँ

**अवलोकन:** एक बार आरंभ हो जाने पर, आप EWS क्लाइंट का उपयोग करके अपने इनबॉक्स में संग्रहीत सभी संदेशों को सूचीबद्ध कर सकते हैं।

#### चरण-दर-चरण प्रक्रिया:

**3.3 क्लाइंट को आरंभीकृत करें (पूर्व आरंभीकरण मानते हुए)**

सुनिश्चित करें कि ग्राहक लिस्टिंग कार्यों के लिए तैयार है।

```java
IEWSClient client = null; // इसे वास्तविक क्लाइंट सेटअप कोड के साथ आरंभ करें
```

**3.4 संदेश पुनः प्राप्त करें और पुनरावृत्त करें**

इनबॉक्स से संदेश प्राप्त करें और आवश्यकतानुसार प्रत्येक संदेश URI को संसाधित करें।

```java
public class ListMessagesFromInbox {
    public static void main(String[] args) {
        try {
            ExchangeMessageInfoCollection msgCollection = 
                client.listMessages(client.getMailboxInfo().InboxUri);

            for (ExchangeMessageInfo msgInfo : msgCollection) {
                String strMessageURI = msgInfo.getUniqueUri();
                // संदेश URI के साथ आगे की प्रक्रिया
            }
        } catch (Exception ex) {
            System.out.println(ex.getMessage());
        }
    }
}
```

**स्पष्टीकरण:**
- `listMessages` निर्दिष्ट मेलबॉक्स URI से सभी संदेशों को पुनर्प्राप्त करता है।
- प्रत्येक के माध्यम से पुनरावृति करें `ExchangeMessageInfo` आगे की कार्रवाई के लिए अद्वितीय URI प्राप्त करने के लिए।

### फ़ीचर 3: संदेशों को मेमोरीस्ट्रीम में सहेजें

**अवलोकन:** संदेशों को मेमोरी स्ट्रीम में सहेजने से आपके जावा अनुप्रयोगों में ईमेल डेटा का कुशल प्रबंधन और प्रसंस्करण संभव हो जाता है।

#### चरण-दर-चरण प्रक्रिया:

**3.5 संदेश URI परिभाषित करें**

वह संदेश निर्दिष्ट करें जिसे आप सहेजना चाहते हैं.

```java
String strMessageURI = "your-message-uri";
```

**3.6 मेमोरीस्ट्रीम में सहेजें**

उपयोग करें `ByteArrayOutputStream` संदेशों को अस्थायी रूप से मेमोरी में संग्रहीत करने के लिए।

```java
public class SaveMessageToMemoryStream {
    public static void main(String[] args) {
        try {
            ByteArrayOutputStream outputStream = new ByteArrayOutputStream();
            client.saveMessage(strMessageURI, outputStream);
        } catch (Exception ex) {
            System.out.println(ex.getMessage());
        }
    }
}
```

**स्पष्टीकरण:**
- `saveMessage` संदेश की सामग्री को प्रदान की गई आउटपुट स्ट्रीम में लिखता है।
- यह दृष्टिकोण डेटा को सीधे डिस्क पर संग्रहीत किए बिना उसे संसाधित करने के लिए उपयोगी है।
