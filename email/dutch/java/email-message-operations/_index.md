## Hoe e‑mailformaten te converteren met Java?
`MailMessage.load` laadt een e‑mailbestand (EML, MSG, enz.) in een MailMessage‑object. `MailMessage.save` schrijft het object naar een nieuw formaat. Om te converteren, roep `MailMessage.load` aan met het bronpad, en roep vervolgens `save` aan met het gewenste uitvoerformaat en de bestemming. De API behandelt codering, bijlagen en metadata automatisch.

## Veelgestelde vragen

**Q: Kan ik een met wachtwoord beveiligd MSG‑bestand converteren naar EML?**  
A: Ja. Laad het bericht met de juiste wachtwoordparameter, roep vervolgens `save` aan met het gewenste formaat; de API ontsleutelt en versleutelt de inhoud automatisch.

**Q: Vereist Aspose.Email for Java dat Microsoft Outlook geïnstalleerd is?**  
A: Nee. De bibliotheek werkt volledig onafhankelijk van Outlook of Exchange Server, waardoor het ideaal is voor batchconversie aan de serverzijde.

**Q: Hoe ga ik om met grote PST‑bestanden zonder het geheugen uit te putten?**  
**`PstReader` biedt een streaming‑API om PST‑bestanden item voor item te lezen.** Gebruik de `PstReader` streaming‑API, die items op aanvraag leest en ze geleidelijk wegschrijft, waardoor het geheugenverbruik onder 100 MB blijft, zelfs voor multi‑gigabyte PST‑bestanden.

**Q: Is er een manier om de oorspronkelijke e‑mailtijdstempels te behouden tijdens conversie?**  
A: Absoluut. Het `MailMessage`‑object behoudt de `DateSent`‑ en `DateReceived`‑eigenschappen, en deze worden automatisch naar het doel­formaat geschreven.
