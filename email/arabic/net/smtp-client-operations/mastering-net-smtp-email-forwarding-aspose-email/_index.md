# كيفية إعادة توجيه رسائل البريد الإلكتروني برمجيًا في .NET باستخدام Aspose.Email SmtpClient

## دليل التنفيذ
### ميزة إعادة توجيه البريد الإلكتروني SMTP
تتيح لك هذه الميزة إعادة توجيه رسائل البريد الإلكتروني مباشرةً باستخدام `SmtpClient` بدون إنشاء يدوي `MailMessage`دعونا نلقي نظرة على عملية التنفيذ.

#### تحديد تفاصيل الخادم وبيانات الاعتماد
ابدأ بتحديد تفاصيل خادم البريد الإلكتروني الخاص بك:

```csharp
string host = "mail.server.com";
int smtpPort = 587;
string username = "username"; // اسم مستخدم SMTP الخاص بك
string password = "password"; // كلمة مرور SMTP الخاصة بك
```

تعتبر هذه المعلمات ضرورية لإنشاء اتصال مع خادم SMTP.

#### تحديد المرسل والمستلمين
حدد من سيرسل البريد الإلكتروني وإلى من يجب إعادة توجيهه:

```csharp
// حدد عنوان البريد الإلكتروني للمرسل
cstring sender = "Sender@domain.com";

// تعريف المستلمين كمجموعة
MailAddressCollection recipients = new MailAddressCollection();
recipients.Add("recepient1@domain.com, recepient2@domain.com");
```

#### إعادة توجيه البريد الإلكتروني
الوظيفة الأساسية هي إعادة توجيه ملف بريد إلكتروني موجود:

```csharp
using (SmtpClient client = new SmtpClient(host, smtpPort, username, password, SecurityOptions.Auto))
{
    // المسار إلى ملف البريد الإلكتروني الخاص بك (تنسيق .eml)
    string fileName = @"YOUR_DOCUMENT_DIRECTORY\test.eml";

    // افتح ملف البريد الإلكتروني للقراءة
    using (FileStream fs = File.OpenRead(fileName))
    {
        // إعادة توجيه البريد الإلكتروني من المرسل إلى المستلمين
        client.Forward(sender, recipients, fs);
    }
}
```

**خيارات تكوين المفاتيح:**
- `SecurityOptions.Auto`:يتم تحديد بروتوكول الأمان تلقائيًا استنادًا إلى إمكانيات الخادم.
- استخدم كتل try-catch حول عمليات الشبكة لمعالجة الأخطاء.

### نصائح استكشاف الأخطاء وإصلاحها
- تأكد من أن تفاصيل خادم SMTP لديك صحيحة ويمكن الوصول إليها من بيئة التطوير الخاصة بك.
- تأكد من أن مسار ملف البريد الإلكتروني دقيق وأن تنسيق الملف صحيح `.eml`.
- تحقق من إعدادات جدار الحماية إذا ظهرت مشكلات في الاتصال.
